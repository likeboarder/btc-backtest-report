一、市場狀態偵測（Bull / Bear Regime）

每根 K 線皆即時判斷當前市場狀態，熊市期間完全暫停開倉，持倉中若轉為熊市則立即平倉。

# 熊市條件（兩者同時成立） trend_bear = EMA_fast < EMA_slow momentum_bear = 近 N 根累積報酬 < 0 regime = 'bear' if (trend_bear and momentum_bear) else 'bull'
EMA 只使用到當前 bar 為止的資料計算，避免前視偏差（Look-ahead Bias）。

二、貝葉斯進場訊號

2-1 似然概率（Likelihood）

取最近 6 根 K 線，從兩個維度提取特徵：

特徵計算方式權重
成交量方向比 上漲 K 線量 ÷ 總量 0.6
K 線實體比 上漲 K 線實體總和 ÷ 總實體 0.4

加權後得多頭似然 L_long，空頭似然 L_short = 1 − L_long，並限制在 [0.1, 0.9]。

2-2 先驗概率（Prior）— 自我更新

初始多空各 0.5，每筆交易結束後動態調整：

  • 各方向保留最近 2 筆歷史損益
  • 勝率高的方向先驗偏高（最多偏離 ±0.47）
  • 多空先驗之和標準化為 1

2-3 後驗概率與開倉判斷

posterior_long = prior_long × L_long （再正規化） posterior_short = prior_short × L_short # 牛市中三個條件同時成立才進場 做多: posterior_long > threshold AND 動量 > 0 AND 成交量 > 均量 做空: posterior_short > threshold AND 動量 < 0 AND 成交量 > 均量

動態門檻調整：

情況門檻變化
高波動市況+最多 0.1（降低誤觸率）
平穩震盪（斜率小 + 低波動）額外 +0.04（避免假突破）
平穩且量能不足再 +0.02

三、動態移動止損（Trailing Stop）

3-1 初始止損

# 槓桿愈高，止損距離愈緊 多倉止損 = 進場價 × (1 − stop_loss_pct / leverage) 空倉止損 = 進場價 × (1 + stop_loss_pct / leverage)

3-2 移動更新邏輯（棘輪機制）

持倉後每根 K 線皆更新止損，止損只往有利方向移動：

# 持倉時間愈長，exp_coef 愈小 → 止損空間拉寬 exp_coef = 0.8 × exp(−0.001 × 已持倉K線數), 最小值 0.2 # 止損回撤比例隨獲利幅度非線性成長 adjusted_stop_pct = base_stop_pct × (1 − exp(−exp_coef × 價格距進場比例)) # 棘輪：只更新對持倉方向有利的方向 多倉新止損 = cur_price × (1 − adjusted_stop_pct) → 只在「高於現有止損」時更新 空倉新止損 = cur_price × (1 + adjusted_stop_pct) → 只在「低於現有止損」時更新
早期持倉止損跟得緊（exp_coef ≈ 0.8）；持倉時間拉長後止損拉開，保留大趨勢的獲利空間。

四、出場條件

觸發原因說明
移動止損被穿破 收盤價 ≤ 止損線（多倉）或 ≥ 止損線（空倉）
市場轉為熊市 當前 K 線判斷為 bear，立即平倉，不等止損觸發

五、策略執行流程

1
Regime 偵測：計算 EMA 快慢線 + 近期動量，判斷牛 / 熊市
2
持倉檢查：熊市或止損觸發 → 平倉並更新先驗概率
3
止損更新：若持倉中且未平倉，更新移動止損至最新位置
4
進場判斷（未持倉）：貝葉斯後驗 + 動量 + 量能 → 達標 → 開多 / 開空
5
重複下一根 K 線

六、參數優化方法

主要超參數透過 Bayesian Optimization + Walk-Forward 驗證 搜尋，優化目標為樣本內夏普比率，以樣本外報酬驗證不過擬合。
